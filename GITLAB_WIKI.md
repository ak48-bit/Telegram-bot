# 线上地推组 - 数据推送系统

## 概述
每日定时推送菲律宾、孟加拉、缅甸三个区域共10个站点的运营数据，包含注册、首存、充值、投产比等10项指标，并自动标记异常站点和风控告警。

## 执行方式
- **手动触发**: Claude Code 内说"更新好了"即可推送最新数据
- **定时任务**: 每小时第7分钟自动执行（Claude Code cron）
- **GitLab CI**: 配置 Pipeline Schedule 后可实现 GitLab 端自动推送

## 数据源
- `26年05月 线上办公数据汇总.xlsx`（按月更新，每月替换路径）

## 输出文件
| 文件 | 语言 | 说明 |
|------|------|------|
| `LATEST_PUSH.md` | 中文 | 实时推送报告 |
| `LATEST_PUSH_EN.md` | English | English report |
| `push_history_*.md` | 中文 | 历史记录（保留最近50份） |

## 覆盖站点
| 区域 | 站点 |
|------|------|
| 菲律宾 (PH) | PH09, PH09-2, PH25, PH18, PH30, PH05, PH16 |
| 孟加拉 (BD) | BD02, BD05 |
| 缅甸 (MM) | MM01 |

## 推送字段
| 字段 | Excel列 | 说明 |
|------|---------|------|
| 总注册人数 | col 7 | 当日注册 |
| 总首存人数 (FTD) | col 8 | 首次存款 |
| 总充值人数 | col 11 | 充值笔数 |
| 总首存金额 | col 20 | 首存金额 |
| 总充值金额 | col 23 | 总充值 |
| 总提款金额 | col 24 | 总提款 |
| 存提差 | col 25 | 充值-提款 |
| 新客单价 | col 15 | 首存金额/FTD |
| 投产比 (ROI) | col 34 | 充值/成本 |
| 状态 | 自动判定 | 🟢/🟡/🔴 |

## 风控规则
| 规则 | 阈值 | 标记 |
|------|------|------|
| 提款率 | >90% | 🚨 |
| 存提差占比 | <10% | 🚨 |
| FTD=0 | =0 | 🔴 |
| FTD个位数 | <10 | 🟡 |
| ROI为负 | <0 | 🟡 |

## GitLab CI 配置
1. 将 `.gitlab-ci.yml` 放入项目根目录
2. 在 GitLab → CI/CD → Schedules 中创建定时任务
3. 设置 cron: `7 * * * *`（每小时第7分钟）
4. 确保数据Excel文件在 Runner 可访问的路径

## Claude Code 配置
- 定时任务已配置为每小时第7分钟
- 用户说"更新好了"时手动触发完整推送
- 脚本路径: `push_update.py` / `push_update_en.py`
