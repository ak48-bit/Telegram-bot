## 缺口3：考核与薪资挂钩 — 优化方案

### 3.1 职级定义（保留原标准，增加提款率红线）

| 职级 | FTD/月 | DIFF/月 | 提款率上限 | 附加条件 |
|------|--------|---------|------------|----------|
| P0 | < 200 | — | — | 新手/试用期 |
| P1 | 200-500 | > 0 | < 90% | 连续 2 月达标 |
| P2 | 500-1000 | > 300K | < 80% | 可带 1-2 新人 |
| P3 | > 1000 | > 500K | < 75% | 可竞聘组长 |

> 新增提款率红线：FTD 再高但如果提款率超标，降一档。

### 3.2 薪资结构建议

```
月收入 = 底薪 + 绩效奖金 + 职级津贴

底薪（按当地标准）
绩效奖金 = DIFF × 提成比例（按职级）
职级津贴（新增）
```

| 职级 | 底薪 | DIFF 提成 | 职级津贴 | 预计月均收入 |
|------|------|-----------|----------|-------------|
| P0 | 基准 | 0.5% | 0 | 基准 |
| P1 | 基准 | 0.8% | +200 | 基准 + 20-30% |
| P2 | 基准 | 1.2% | +500 | 基准 + 50-70% |
| P3 | 基准 | 1.5% | +800 | 基准 + 80-120% |

> 示例：P2 业务员月 DIFF=400K → 底薪 + 400K×1.2% + 500 = 底薪 + 4,800 + 500

### 3.3 晋升机制

| 条件 | 操作 |
|------|------|
| 连续 2 月达到上级标准 | 自动晋升 |
| 当月提款率超红线 | 暂停晋级，警告 1 次 |
| 连续 2 月不达标 | 降 1 级 |
| 新人入职 | 统一 P0，2 月后首评 |

### 3.4 考核日历

```
每月 1 日：拉上月数据，出职级评定
每月 3 日：公布上月职级变更名单
每月 5 日：发薪时体现新职级
```

## 缺口5：防作弊/data造假 — 4 项核查指标

### 5.1 四项红线指标

| 指标 | 计算方式 | 红线 | 触发动作 |
|------|----------|------|----------|
| **注册→首存转化率过高** | FTD ÷ 注册数 | > 70% | 怀疑虚拟号/自注册 |
| **存款=提款（对倒）** | 存提差=0 的玩家人数 | > 总FTD 5% | 怀疑刷流水 |
| **存提差占比太低** | DIFF ÷ DPS | < 10% | 怀疑套利/高赔付 |
| **同金额充值占比过高** | 相同充值金额出现次数 | > 总充值笔数 20% | 怀疑机器刷单 |

### 5.2 每周抽查流程

```
每周五下午：
1. 从每个站点随机抽 3 个新注册玩家
2. 检查：注册时间、首存时间、首存金额、提款金额
3. 标记异常：
   - 注册后 5 分钟内首存 → 🟡 可疑
   - 首存后 1 小时内提款 → 🔴 高危
   - 存提金额完全一致 → 🔴 套利
4. 汇总到质检表，周一早会通报
```
